# 商品期货主力连续合约拼接管线 — 构造规则

本文件定义全天候商品 sleeve 所需的**主力连续合约**如何从 Tushare 原始数据构造。它是
[商品期货 Sleeve 设计方案](../etf-all-weather-implementation/commodity-futures-sleeve-design.md) §4.1 的展开，也是整个长历史回测正确性的根基。本文件是规则说明与验收标准，不是实现代码。

规则中的数据形态均来自对项目现有 Tushare token 的实测（`tushare==1.4.24`），实测细节见主设计方案 §6。

## 1. 为什么必须自己拼

Tushare 直接提供的"主力连续日线"（`fut_daily(ts_code="M.DCE")` 这类连续根）**只有 2024-01 起约 614 个交易日**，不足以支撑全天候的长历史回测。深历史（豆粕/铜/铝到 2005 年）只能通过两条深历史数据自行拼接：

- `fut_mapping`：point-in-time 的每日主力合约映射（深历史，2005–2018 起）。
- `fut_daily` 单合约：每个具体合约（如 `M2609.DCE`）的完整逐日 OHLC/settle/vol/oi（深历史）。

拼接的本质：**按 `fut_mapping` 逐日选出当日主力合约，取该合约当日价格，在换月边界做后向复权，串成一条连续序列。**

## 2. 输入数据（实测形态）

| 数据 | 接口 | 关键字段 | 用途 |
| --- | --- | --- | --- |
| 主力映射 | `fut_mapping(ts_code=<root>)` | `trade_date`, `mapping_ts_code` | 逐日主力合约 = 拼接锚点 |
| 单合约日线 | `fut_daily(ts_code=<contract>)` | `open/high/low/close`, `settle`, `pre_settle`, `vol`, `oi`, `oi_chg` | 价格来源 + 主力交叉校验 |
| 合约元数据 | `fut_basic(exchange=<ex>)` | `multiplier`, `per_unit`, `trade_unit` | 名义价值换算 |

连续根代码（实测正确后缀，`.SHF/.DCE/.ZCE/.INE`，非交易所全称）：
`AU.SHF / AL.SHF / CU.SHF / RB.SHF / I.DCE / M.DCE / P.DCE / SC.INE / TA.ZCE`。

**注意**：`fut_daily` 单合约含 `vol` 与 `oi`，因此"谁是主力"可以自己算（见 §5 交叉校验），不必只信 `fut_mapping`。

## 3. 换月的实测特征（决定算法的三个事实）

以豆粕 `M.DCE` 2026-04-09 的一次真实换月为例（`M2605.DCE → M2609.DCE`）：

1. **换月是离散跳变**：`fut_mapping` 在某一交易日整体从旧合约切到新合约，不是渐变。豆粕主力约每 4 个月换一次（1/5/9 月合约轮动），节奏规律但**因品种而异**（黑色/能化换月更频繁）。
2. **换月当日两个合约价差很大**：换月日 `M2605` 收 2847、`M2609` 收 2971，**同一天差 +124 点 = +4.36%**。这是纯粹的近远月基差，不是收益。
3. **持仓量交叉可交叉验证主力转移**：换月前后旧合约 `oi` 从 102 万降到 66 万，新合约从 168 万升到 199 万。持仓量交叉与 `fut_mapping` 的切换日应一致。

**核心结论**：一次**不做后向复权的天真拼接会凭空注入 +4.36% 的假单日涨幅**。9 个品种、十几年、每年数次换月，这些假跳空会严重污染回测的收益、波动率和最大回撤。后向复权不是可选优化，是正确性前提。

## 4. 拼接规则

### 4.1 主力选择（point-in-time）

- 连续序列第 `t` 日的价格 = `fut_mapping` 在第 `t` 日给出的 `mapping_ts_code` 对应合约在第 `t` 日的价格。
- **PIT 边界**：只能使用第 `t` 日及之前已知的映射，绝不能用未来才知道的主力信息回填历史。换月切换在 `fut_mapping` 的 `trade_date` 当日生效，连续序列在该日即切换到新合约。
- 换月日的处理：换月日**当日**已用新合约价格；后向复权只调整换月日**之前**的历史段（见 4.2），换月日及之后不动。

### 4.2 后向复权（back-adjustment）

采用**后向复权（back-adjust 老段）**，保持最新价格为真实市场价（便于对齐当前可交易价、名义价值换算）：

- 在每个换月日计算调整量。两种口径二选一，需在 Phase 0 冻结：
  - **差值法（推荐）**：`adj = 新合约换月日价 − 旧合约换月日价`，把换月日之前的整段历史价格全部加上累计 `adj`。加法保持价差绝对量，适合大多数品种。
  - **比值法**：`ratio = 新价 / 旧价`，历史段整体乘以累计 `ratio`。乘法保持百分比收益，但可能使深历史价格失真为负或极端值，需评估。
- **换月日选取用哪个价**：用 `close` 还是 `settle`（结算价）需冻结。期货以 `settle` 结算，`settle` 通常更稳；实测两者在换月日都可得。建议用 `settle` 计算 `adj`，用 `close`/OHLC 构造序列，口径在 Phase 0 固定并写入 artifact。
- 复权后的连续序列**只用于计算收益率**，其绝对价位是复权价，不代表任何单一合约的真实价——文档与看板须标注这一点（对照 [ETF 收益语义说明](./etf-return-semantics-note.md) 对收益语义的区分要求）。

### 4.3 输出

连续序列每行至少包含：`trade_date`, 复权后 `open/high/low/close`, 复权后 `settle`, 原始 `close`(未复权，供审计), `active_contract`(当日主力 ts_code), `is_roll_day`(bool), `roll_adj`(当日累计复权量), `vol`, `oi`。落到 lakehouse（沿用 ETF 日线的 parquet 惯例）。

## 5. 交叉校验（不只信 fut_mapping）

因 `fut_daily` 带 `vol`/`oi`，构造后须做一致性检查，任何一项不通过则该品种该区间标记为不可用、进入人工复核：

- **主力一致性**：`fut_mapping` 给出的主力，与"当日持仓量 `oi` 最大的合约"应大体一致。持续不一致说明映射口径或数据有问题。
- **换月连续性**：每个 `is_roll_day` 前后，旧合约 `oi` 应下降、新合约 `oi` 应上升（实测的持仓量交叉）。
- **无未来数据**：复权只改换月日之前的历史；换月日及之后价格必须等于原始单合约价。用未复权 `close` 列可审计这一点。

## 6. 明确不做

- 不做**前向复权**（会改变最新价，破坏名义价值口径）。
- 不引入**加权连续/等权连续/展期加权**等复杂连续合约口径——第一版只做"主力连续 + 后向复权"，最简且可解释。
- 不在拼接层引入**杠杆**（保证金话题见主方案 §4.3）。
- 不用 Tushare 的 2024-01 起短连续日线作为长历史回测来源（只可用作交叉核对近段）。

## 7. 验收标准

Phase 1 该管线通过的判据：

1. 9 个品种的连续序列可从 `fut_mapping` + `fut_daily` 单合约完整重建，且可重复（同输入同输出）。
2. 复权口径（差值/比值、close/settle）已在 Phase 0 冻结并写入 artifact。
3. 换月日集合与 `fut_mapping` 切换日一致；每个换月日通过 §5 的 `oi` 交叉检查。
4. 抽查若干换月日：复权后连续序列在换月日**无**基差型假跳空（对照未复权列，假跳空应被吸收）。
5. point-in-time 检查通过：换月日及之后的价格等于原始单合约价，历史段仅被后向调整。
6. 与 Tushare 2024-01 起的短连续日线在重叠区间做形状核对，收益率量级一致（不要求逐点相等，口径可能不同）。

未通过任一项的品种，不进入 Phase 2 的商品篮子合成。
