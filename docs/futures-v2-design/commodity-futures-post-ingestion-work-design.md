# TradePilot 商品期货接入到研究结论总设计

## 1. 文档目标

本文定义商品期货从基础数据接入验收，到形成可审计的商品 `sleeve` 研究结论之前的统一工作顺序、交付物和阶段门槛。因此“接入后”表示主数据链路已经可运行，不表示数据质量、连续合约和研究口径已经完成验收。

本文是 V2 的流程编排文档，不替代 ETF 全天候 V1 的既有实现细节。V1 聚焦文档是历史基线；V2 在需要时复用并扩展它们，若两者冲突，以本文件定义的阶段门槛和当前代码实际实现为准。V2 不再维护第二份学习指南，`commodity-futures-learning-guide.md` 作为唯一学习入口。

本阶段的目标不是生成真实期货订单，而是回答一个更窄的问题：在不引入未来数据、虚假换月收益和隐含杠杆的前提下，商品期货能否为现有 ETF 全天候组合提供稳定、可解释、成本后仍成立的独立风险来源。

候选池沿用 `AU / AL / CU / RB / I / M / P / SC / TA`。其中 AU 作为现有黄金 ETF 暴露的对照，默认不进入非黄金商品篮子。

## 2. 范围与边界

### 2.1 本设计包含

- 接入数据的质量验收与冻结。
- 单合约数据语义和单位校验。
- point-in-time 主力映射与连续合约构建。
- 换月、复权和收益口径审计。
- 商品篮子候选筛选、权重和风险贡献研究。
- 与现有 ETF 全天候基线的增量回测。
- 手续费、冲击和换月成本敏感性分析。
- 研究结论、失败样本和后续决策记录。

### 2.2 本设计不包含

- 实盘或模拟盘下单。
- 经纪商接口、报单和成交回报。
- 保证金账户、逐日盯市和强平处理的完整实现。
- 日内策略、择时交易或高杠杆优化。
- 在研究结论形成前改造现有 regime、调仓或执行系统。

连续合约收益不能直接转为订单。进入模拟盘前，仍需单独设计具体合约选择、开平仓语义、整数手、乘数、保证金、结算、移仓和交易成本。

## 3. 启动条件

只有数据接入满足以下条件，才开始后续研究：

1. `fut_basic`、单合约 `fut_daily` 和 `fut_mapping` 已落库，且每次同步有 ingestion run 记录。
2. 候选品种、交易所后缀、合约代码和交易日字段已经标准化。
3. 原始数据可追溯到来源、拉取时间和批次，重复同步具有幂等性。
4. `open/high/low/close/settle/pre_settle/vol/oi/oi_chg` 的来源单位已明确记录，不做隐式换算。
5. 数据缺口、接口历史起点和已知源限制已有清单。

若任一条件不满足，工作停留在数据接入阶段，不构造组合回测。

## 4. 总体工作流

```text
接入验收
  -> 单合约质量基线
  -> 主力映射与连续序列
  -> 单品种收益审计
  -> 商品篮子研究
  -> ETF 基线增量回测
  -> 成本与稳健性评估
  -> 接受 / 观察 / 拒绝
```

每一阶段必须保留可复算的数据或报告。下游不能只消费一条处理后的净值曲线，而无法追溯原始合约和换月来源。

## 5. 阶段 0：接入验收与数据冻结

### 5.1 工作内容

- 统计每个品种的合约数量、日期范围、交易日覆盖率和重复键。
- 检查 OHLC 关系、负值、空值、异常跳变和结算价缺失。
- 检查 `vol` 与 `oi` 的单位和缺失模式，不跨来源猜测单位。
- 检查 `fut_mapping` 指向的合约在对应交易日是否存在行情。
- 区分自然日和交易日，确认夜盘数据归属口径。
- 固定研究数据快照、参数版本和代码版本。

### 5.2 交付物

- 字段与单位清单。
- 品种级覆盖率报告。
- 缺口与异常记录。
- 可复现的数据快照标识。

### 5.3 通过门槛

- 主键无重复，核心价格字段不存在未解释的系统性缺失。
- 主力映射能够关联到实际单合约行情。
- 每个异常都有修复、排除或接受理由。
- 任一收益结果都能追溯到固定的数据快照。

## 6. 阶段 1：先审计单合约与一次换月

### 6.1 工作内容

先选择豆粕 M 作为最小样本，不同时铺开九个品种：

1. 用 `fut_basic` 的交易单位计算一手名义价值和价格变动 1% 的盈亏。
2. 选择一次完整主力切换，观察新旧合约前后至少 5 个交易日。
3. 对照新旧合约的 `close`、`settle`、`vol`、`oi` 和主力映射。
4. 计算天真拼接产生的假跳空，并确认其来源是合约切换而非市场单日收益。

### 6.2 交付物

- 单合约计算样例。
- 换月窗口审计表。
- 主力切换和假跳空解释。

### 6.3 通过门槛

- 计算结果可以由原始字段复算。
- 主力切换只使用当时可知的数据，不使用事后规则回填。
- 能明确区分价格变化、换月价差和真实交易成本。

## 7. 阶段 2：构建可审计的连续合约

### 7.1 数据产物

连续序列至少保留：

| 字段 | 含义 |
| --- | --- |
| `trade_date` | 交易日 |
| `root_symbol` | 连续根代码 |
| `active_contract` | 当日实际主力合约 |
| `raw_close` | 主力合约原始收盘价 |
| `raw_settle` | 主力合约原始结算价 |
| `adjusted_close` | 后向复权连续价格 |
| `continuous_return` | 连续收益 |
| `is_roll_day` | 是否换月 |
| `roll_from` / `roll_to` | 新旧合约 |
| `roll_gap` | 换月调整量 |
| `cumulative_roll_adjustment` | 累计复权量 |
| `source_run_id` | 来源接入批次 |

### 7.2 构建原则

- `active_contract` 优先遵循冻结的 `fut_mapping`，`vol` 和 `oi` 只做交叉验证。
- 复权使用预先冻结的公式，查看回测结果后不得切换算法。
- 后向复权仅用于研究收益，不把复权后的绝对价格解释为可交易合约价格。
- 保留天真拼接序列用于诊断，但禁止用于绩效计算。
- 每次换月必须能定位新旧合约价格和调整量。

### 7.3 验收

- M 的完整序列先通过人工抽样，再扩展其他品种。
- 随机抽取普通交易日和换月日，收益均可由保存字段复算。
- 换月日不存在由合约价差直接制造的异常收益。
- 同一输入快照和参数重复运行得到一致结果。

## 8. 阶段 3：单品种质量与可研究性筛选

对每个品种分别生成质量卡，至少包括：

- 有效历史长度、缺失率和连续性。
- 主力切换次数、异常换月次数和平均持有周期。
- 成交量、持仓量和流动性异常。
- 收益分布、波动率、最大回撤和极端日。
- 与其他候选、现有黄金 ETF 和现有 sleeves 的相关性。
- 一手名义价值与假设组合规模的整数手误差，仅作为未来可交易性提示。

品种结论只能是：

- `accept`：数据和风险来源均支持进入篮子研究。
- `observe`：可研究但历史、流动性或独立性证据不足。
- `reject`：数据无法审计、风险暴露重复或成本后没有研究价值。

AU 默认作为对照；若纳入正式篮子，必须证明改善并非重复现有黄金 ETF 暴露。

## 9. 阶段 4：商品篮子设计

### 9.1 初始候选

第一轮使用非黄金候选：

```text
AL / CU / RB / I / M / P / SC / TA
```

先比较两种简单规则：

- 等权：用于观察各品种直接贡献。
- 等风险：避免高波动品种自然主导篮子。

在看到绩效前冻结调仓频率、波动估计窗口、权重上限和缺失数据规则。不以优化历史收益为目标。

### 9.2 风险约束

- 检查 AL/CU、RB/I、SC/TA 的滚动相关性和板块风险贡献。
- 同产业链品种不能仅因代码不同就视为独立风险来源。
- 报告名义权重、波动贡献和边际风险贡献。
- 做剔除单品种、剔除单板块和纳入/剔除 AU 的敏感性分析。

### 9.3 交付物

- 候选接受/观察/拒绝表。
- 等权与等风险篮子定义。
- 板块和品种风险贡献报告。
- 权重规则和参数冻结记录。

## 10. 阶段 5：接入现有 ETF 全天候基线

保持现有 ETF 基线、regime 和其他参数不变，只增加一个商品 `sleeve`，避免同时修改多个变量而无法归因。

至少比较：

- 原有基线与“基线 + 商品”的年化收益、波动率、最大回撤和收益回撤比。
- 商品的收益贡献与总风险贡献。
- 不同宏观区间、滚动窗口和 walk-forward 窗口的稳定性。
- 等权与等风险商品篮子的方向一致性。
- AU 纳入与剔除、单品种和单板块剔除后的结果。

第一轮只回答商品暴露是否值得保留，不据此立即改造 regime 或执行系统。

## 11. 阶段 6：成本和稳健性评估

### 11.1 成本情景

至少覆盖：

- 开平手续费。
- 买卖冲击或滑点。
- 换月移仓成本。
- 不同调仓频率。
- 成本参数上调后的压力情景。

成本参数必须标注来源和日期；无法 point-in-time 化的成本只作为敏感性假设，不包装成历史精确值。

### 11.2 稳健性检查

- 缩短或平移样本区间。
- 改变合理的波动估计窗口。
- 比较等权和等风险结果。
- 检查收益是否集中于少数年份、单一品种或单一危机区间。
- 保存失败窗口和不支持假设的样本。

### 11.3 最终通过门槛

商品 `sleeve` 只有同时满足以下条件才进入下一阶段：

1. 连续序列通过换月和 point-in-time 审计。
2. 成本后仍改善组合，且改善不依赖单一品种或短区间。
3. 风险贡献与目标权重相符，没有隐含板块集中。
4. AU 对照证明改善不只是重复黄金暴露。
5. 结果在等权、等风险和合理参数扰动下方向大体一致。
6. 失败样本、限制和数据短历史被明确披露。

不满足时应选择 `observe` 或 `reject`，而不是继续增加参数寻找通过结果。

## 12. 决策与后续分支

### 12.1 接受

若研究通过，下一份设计应聚焦模拟盘准备，包括：

- 具体合约和换月执行规则。
- 目标名义暴露到整数手的转换。
- 开仓、平仓、多头和空头语义。
- 保证金、现金缓冲和逐日盯市。
- 手续费、滑点、涨跌停和无法成交处理。
- 纸面订单、成交回放和账户对账。

这些内容不应塞回连续合约研究模块。

### 12.2 观察

记录缺失证据和复查条件，例如积累更多历史、改善数据源或等待足够的前向观察期。观察状态不进入组合默认配置。

### 12.3 拒绝

保留拒绝原因、复算产物和失败报告。拒绝某个品种或整个商品 `sleeve` 是有效研究结论。

## 14. 第一批执行顺序

下一步按以下顺序开始，不并行扩展范围：

1. 生成九个候选品种的数据覆盖率和字段单位报告。
2. 用 M 完成一个换月窗口人工审计。
3. 用 M 完成可追溯的连续序列，并通过复算。
4. 扩展到其余品种，逐一给出 `accept/observe/reject`。
5. 冻结商品篮子规则，再运行增量回测和成本敏感性分析。
