# ETF 全天候策略优化阶段计划

## 1. 本阶段结论先行

本阶段只做两件事：

1. 参数优化回测。
2. 新标的候选调研。

阶段结束后输出一份项目总结报告。

不在本阶段做：

- 不直接进入实盘。
- 不一次性重写优化器。
- 不把所有新标的一起加入组合。
- 不用历史最优参数直接替代当前版本。

## 2. 当前基线与问题

当前 V1 使用 5 个 frozen sleeves：

| Sleeve | ETF | 主要风险暴露 |
| --- | --- | --- |
| `equity_large` | `510300.SH` | A 股大盘权益 |
| `equity_small` | `159845.SZ` | A 股小盘权益 |
| `bond` | `511010.SH` | 债券防御 |
| `gold` | `518850.SH` | 黄金与压力对冲 |
| `cash` | `159001.SZ` | 现金与短久期缓冲 |

现有链路为：

```text
market / macro context
-> regime score
-> risk budget
-> budgeted inverse-vol target weight
-> monthly frozen weight
-> common backtest kernel
-> robustness / shadow evaluation
```

截至基线报告所覆盖的 2025-01 至 2026-05，共有 17 个调仓周期：

| 指标 | 当前动态策略 | `static_inverse_vol_v1` |
| --- | ---: | ---: |
| 累计收益 | 21.49% | 21.43% |
| 年化波动率 | 6.65% | 6.54% |
| Sharpe | 2.44 | 2.48 |
| 月均换手 | 5.03% | 4.34% |
| 最大回撤 | -4.83% | -4.82% |

当前证据支持以下判断：

- 动态策略尚未相对静态 inverse-vol 显示稳定增益，且换手更高。
- 85 行 risk budget 中约 75 行为 `partial`；`confidence cap = 0.35` 使多数主动 tilt 接近中性预算。
- 当前样本较短，不足以证明动态信号长期无效，也不足以支持继续增加复杂度。
- `cash` 的近零波动率可能扭曲 inverse-vol 权重；大小盘可能主要重复同一个 A 股 beta。
- 当前 kernel 的日度权重行为与月度 turnover 记录口径仍需对照验证。

因此，本阶段不预设“调参后必然优于基线”或“新增标的必然改善组合”。若证据不足，保留当前简单基线也是有效结论。

## 3. 本阶段交付物

### 3.1 参数优化

交付：

- 参数清单。
- 批量回测脚本。
- walk-forward 结果。
- 参数邻域和稳定性结果。
- 是否保留当前基线的判断。

### 3.2 新标的调研

交付：

- 纳斯达克 ETF 候选表。
- 大宗商品 ETF/基金候选表。
- 豆粕基金候选表。
- 每个候选的进入、拒绝或继续观察理由。

候选调研只形成短名单。具体产品代码以正式基金资料、交易所信息和可获得数据为依据，不凭印象提前冻结。

### 3.3 阶段总结报告

交付：

- 研究问题。
- 数据与方法。
- 实验结果。
- 结论。
- 局限与后续计划。

报告按实际实验产出组织，不在实验前固定更细的论文目录。

## 4. 执行顺序

### Phase 0：冻结基线与实验合同

冻结数据区间、基准策略、回测内核、成本口径、turnover 口径和实验结果最小字段。

### Phase 1：参数批量回测

整理参数清单，编写批量脚本，完成 walk-forward、参数邻域和成本敏感性对比，形成是否保留当前基线的判断。

### Phase 2：候选标的调研

参数实验形成初步结论后，再整理纳斯达克、大宗商品和豆粕基金候选短表，不在此阶段接入全部候选。

### Phase 3：单变量组合对照

只对通过初筛且数据可用的标的逐个测试；每次只增加一个风险暴露，避免与参数变化混杂。

### Phase 4：阶段总结报告

汇总实际完成的实验、候选调查、失败项和限制，形成可追溯的项目总结。

## 5. 立即待办

1. 冻结当前基线的数据区间、回测内核、成本和 turnover 口径。
2. 整理首轮可调整参数及候选范围。
3. 定义批量实验所需的最小输入和结果字段。
4. 编写批量回测脚本并先跑通一个小规模参数组合。
5. 在小规模实验可重复后，启动完整参数矩阵和 walk-forward 回测。

Phase 2-4 不属于立即待办；它们在 Phase 1 形成可复核结果后依次启动。

## 6. 方法与约束附录

本节是执行参考，不是当前待办清单。具体约束可在 Phase 0 根据现有代码和数据能力进一步冻结，但不得降低 point-in-time、可追溯性和单变量对照要求。

### 6.1 参数研究范围

首轮优先测试当前模型中已有经济含义的参数：

| 参数组 | 候选参数 | 首轮范围 | 主要观察 |
| --- | --- | --- | --- |
| 风险估计 | 波动率窗口 | 20、60、120、252 个交易日 | 权重稳定性、回撤、换手 |
| 风险估计 | 多窗口组合 | 20/60、60/120、60/252 | 是否降低单窗口敏感性 |
| 风险预算 | 单 sleeve 权重上限 | 当前值及少量相邻档位 | 集中度与约束命中率 |
| 动态预算 | `confidence cap` | 当前值及少量相邻档位 | tilt 是否产生有效增量 |
| 动态预算 | 最大 tilt 幅度 | 保守、中性、积极三档 | 主动贡献与换手 |
| 趋势规则 | 均线窗口 | 40、60、80、120 日 | 简单趋势能否替代复杂 regime |
| 调仓规则 | 调仓频率 | 月度、季度 | 成本与响应速度 |
| 调仓规则 | 单 sleeve 偏离阈值 | 2%、3%、5% | 无效交易减少量 |

实验按以下顺序分组，不同时改变多个主要因素：

1. 固定数据和 kernel，比较 cash 是否退出风险优化、大小盘是否合并为权益风险桶。
2. 在选定的静态 universe 上比较波动率窗口。
3. 比较静态预算、简单趋势和当前 regime 链条。
4. 独立比较调仓频率和偏离阈值。

首轮不引入深度学习、强化学习、黑盒 alpha forecast 或大规模无约束网格搜索。

### 6.2 批量回测与选择原则

批量脚本的最小流程为：

```text
读取实验参数
-> 生成独立 frozen artifact
-> 调用统一 backtest kernel
-> 汇总指标、diagnostics 和 artifact 引用
```

至少记录实验标识、策略版本、参数、数据区间、运行状态、失败原因和核心指标。失败候选不能被静默删除，历史排名第一的参数不能自动写入当前版本。

最终判断以 walk-forward 和参数邻域为主。候选应在相邻参数、多个时间切分和合理成本下保持结论方向一致；只有一个精确参数点有效时，不应晋级。

### 6.3 新标的调查与初筛

候选表只保留支持初筛所需的字段：

- 产品名称、代码、产品类型和交易场所。
- 跟踪指数或底层资产。
- 成立或上市时间及可用历史。
- 管理费、托管费和交易或申赎费用。
- 流动性、限购、额度或暂停申赎约束。
- 汇率、展期、基差、跟踪误差等主要风险。
- 与现有 sleeves 的预期差异及进入、拒绝或观察理由。

增加标的是为了补充独立风险来源，不是扩大数量。通过初筛至少意味着：风险暴露与现有 sleeves 有可解释差异、数据可追溯、主要交易限制可识别。

场内 ETF 与场外基金不能共用同一成交假设。场内 ETF 主要考虑市场价格、成交量、价差和折溢价；场外基金主要考虑未知净值、申赎确认时滞、持有期规则和费用。场外豆粕基金若无法可靠建模这些条件，只作为研究候选，不进入可交易组合回测。

### 6.4 数据质量与评价口径

所有特征、状态和权重只能使用决策时点之前已可用的数据。ETF 上市前若使用指数或其他代理，必须区分代理区间与真实可交易区间。

最低限度检查交易日缺口、停牌或零成交、复权异常、代理拼接边界和未来数据。海外或商品候选还需检查交易日历、时区、汇率换算、展期和基差；场外基金需检查净值可用时点及申赎延迟。

统一报告累计收益、年化收益、年化波动率、最大回撤、Sharpe、turnover 和成本后收益。参数或新标的的价值还应通过收益贡献、风险贡献及回撤变化解释。

### 6.5 停止条件

出现以下情况时，候选停止或返回重新设计：

- 使用未来数据、事后修订值或事后挑选的有利区间。
- 仅全样本有效，walk-forward 无效。
- 主要结论在轻微参数扰动后反转。
- 成本后优势消失。
- 新标的与现有 sleeve 高度重复，或改善只来自集中押注。
- 海外、商品或场外基金的关键交易限制无法建模。
- 结果无法从参数、数据版本和 frozen artifact 重建。
- 复杂候选相对简单基准没有明确增益。

### 6.6 阶段总结报告参考

报告只需回答五个问题：

1. 本阶段研究了什么问题。
2. 使用了什么数据、基线和实验方法。
3. 参数实验和候选标的调查得到了什么结果。
4. 当前应保留、拒绝或继续观察什么。
5. 结论有哪些限制，下一阶段应做什么。

报告至少附上参数清单、运行命令、结果索引和失败实验。通过以下 checklist 即可验收：

- 参数矩阵已按冻结口径完成或明确记录未完成原因。
- walk-forward 结果可以重复生成。
- 参数选择或保留基线的理由可由结果支持。
- 每个新标的候选都有可追溯的初筛结论。
- 报告中的主要结论能定位到对应 artifact 或运行记录。
