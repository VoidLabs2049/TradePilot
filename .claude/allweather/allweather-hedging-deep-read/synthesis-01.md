# Synthesis 01 — AllWeather Hedging Deep Read: 最终判断

Status: **complete**

## 一句话总结

> `all_weather_hedging/` 是一套骨架扎实但灵魂缺失的演示代码：它有可工作的 CNN + 风险平价 + 期权对冲管线，但没有全天候策略最核心的宏观状态识别层。我们的框架正好相反——有完整的宏观识别理论但无可运行代码。两者**正交互补**。

## 稳定结论

### 1. 他们的工程骨架可以直接继承 4 个模块
- `risk/parity.py` — Newton 风险平价求解器
- `risk/engine.py` — VaR/CVaR/MRC/RC 计算
- `hedging/options.py` — BS Greeks 引擎
- `data/schema.py` — 清晰的数据 schema 设计

### 2. 他们最大的技术贡献：多任务 CNN 先验
`AllWeatherRisk1DCNN` 从收益序列同时预测 vol/corr/tail 的设计是合理的，但它不应该替代宏观状态识别——它应该是状态识别层**之下**的统计矩估计器：
```
宏观变量 → 状态识别 → 风险预算
收益序列 → CNN → vol/corr/tail (给 ERC 提供更准确的协方差矩阵)
```

### 3. 我们最应该紧迫补齐的：状态识别层
没有这一层，任何风险平价实现都只是"固定预算的风险平价"，不是真正的"全天候"。这是灵魂。

### 4. 期权对冲层值得保留但需要改造
从"对每只股票独立对冲"改为"对4类宏观资产做组合层尾部保护"，从合成 BS 价格改为考虑真实市场的买卖价差和偏度。

### 5. 合成数据是演示骨架，不是研究基础
所有模型（CNN 预训练、期权定价、数据流）都依赖合成数据。要进入实战，必须建立真实数据接口（Tushare宏观 + ETF价格）。
