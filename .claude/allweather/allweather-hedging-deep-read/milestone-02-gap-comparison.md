## 逐层详细对比

### Layer 1: 资产池定义

| 维度 | 我们 | 他们 |
|------|------|------|
| 资产数量 | 6-7个bucket（权益/债券/黄金/商品/CTA/现金/海外） | 4类宏观资产 + 50只合成股票 |
| ETF映射 | 有具体A股ETF（沪深300/中证500/国债/黄金ETF等） | 纯合成ticker (STK_000~STK_049) |
| 中国约束 | ETF可选池、流动性、政策风险 | 无 |
| 判断 | **我们的资产池更贴近实战** | 他们是演示级 |

### Layer 2: 特征工程

| 维度 | 我们 | 他们 |
|------|------|------|
| 宏观变量 | PMI/PPI/M1-M2/社融/信用利差（15+指标） | 无 |
| 市场变量 | 风格相对强弱/融资余额/成交扩散 | 无 |
| 技术变量 | MA/ADX/RSI/ATR | 无 |
| 输入形式 | 多维特征向量 | 4维收益序列 (64步窗口) |
| 判断 | **我们有完整的特征框架** | 他们只用价格序列，丢掉了大量信号 |

### Layer 3: 状态识别 ← 核心缺口

| 维度 | 我们 | 他们 |
|------|------|------|
| 方法 | 规则打分 / 逻辑回归 baseline | ❌ 缺失 |
| 输出 | 象限分布 P(Q=q\|X) + 置信度 | ❌ 无 |
| 平滑 | 时序一致性，避免跳变 | ❌ 无 |
| 判断 | **这是两者之间最大的结构洞** | 他们的CNN预测的是vol/corr/tail，不是regime |

**关键洞察**: 他们的CNN做的是 `returns → vol/corr/tail`（统计矩估计），我们需要的状态识别是 `macro variables → regime → risk budget`（经济状态→配置决策）。这两个任务**正交互补**:
- 统计矩估计告诉你在给定环境下资产的风险特征
- Regime识别告诉你当前处于什么环境，该承担多少风险

**最佳的融合方式**:
```
宏观变量 → [状态识别器] → 风险预算(b_i)
                              ↓
收益序列 → [CNN] → vol/corr/tail
                              ↓
                    ERC优化: min Σ(RC_i - b_i)²
```

### Layer 4: 风险预算

| 维度 | 我们 | 他们 |
|------|------|------|
| 预算来源 | 象限+置信度驱动，随环境变化 | 固定 [0.4, 0.3, 0.15, 0.15] |
| 中性预算 | 有明确定义 | 无 |
| 倾斜幅度 | 置信度控制：c*Δb_i | 无 |
| 判断 | **我们的预算更合理** | 固定预算 = 全天候的退化版本 |

### Layer 5: ERC优化

| 维度 | 我们 | 他们 |
|------|------|------|
| 公式 | min Σ(RC_i-b_i)² | 同左 |
| 方法 | 概念描述 | Newton法实现 ✓ |
| 约束 | 单资产上限/换手率/流动性/风格暴露 | 仅非负约束 |
| 判断 | 他们有**可工作的求解器** | 我们的约束更全但未实现 |

### Layer 6: 再平衡

| 维度 | 我们 | 他们 |
|------|------|------|
| 频率 | 月度+阈值触发 | Streaming (tick级，无实际再平衡逻辑) |
| 触发条件 | 风险贡献偏离预算超阈值 | 每个chunk都重新计算权重 |
| 成本 | 考虑交易成本和换手率 | 期权层有交易成本，配置层无 |
| 判断 | 我们的再平衡纪律更适合配置场景 | 他们是演示级实时流 |

### Layer 7: 回测验证

| 维度 | 我们 | 他们 |
|------|------|------|
| 框架 | 样本外验证/参数敏感性/不同市场阶段 | ❌ 缺失 |
| 判断 | 需要做 | 需要做 |

### Layer 8: 期权对冲 ← 他们有，我们完全没有

**他们新增的一层**：用虚值期权来对冲尾部风险。

这一层在我们的 ETF AllWeather 框架中完全没有讨论。这是有意义的补充方向——全天候的核心理念是"在各种宏观环境下生存"，而期权对冲正是尾部风险管理的标准工具。

但他们的实现方式有问题:
- 对冲每只股票而非宏观组合 → 应该对冲4类宏观资产的尾部风险，而非50只股票
- 没有考虑对冲成本对组合收益的长期侵蚀（期权费率）
- 没有对冲策略的回测评估

## 最关键的判断

> `all_weather_hedging/` 是一套**技术演示**而非**实战系统**。它的工程骨架（模块化管线、CNN模型、风险平价求解器、期权对冲）是扎实的，但缺少了全天候策略的灵魂——**宏观状态识别驱动的动态风险预算**。

我们的框架有灵魂但缺骨架。他们的代码有骨架但缺灵魂。

**融合路径**:
1. 保持他们的工程骨架（parity/engine/schema/options）
2. 新增状态识别层（我们的框架）
3. 让状态识别层输出风险预算，驱动他们的parity优化器
4. 改造期权对冲从"个股独立"到"宏观组合层"
5. 全部接入真实数据
