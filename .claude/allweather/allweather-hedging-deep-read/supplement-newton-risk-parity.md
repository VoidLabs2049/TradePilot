## 一、风险平价优化的数学形式

风险平价要求各资产对总风险的贡献相等。

设组合权重 **w** ∈ ℝⁿ，协方差矩阵 Σ，则边际风险贡献：

```
MRC_i = ∂σ_p/∂w_i = (Σw)_i / σ_p
```

风险贡献：
```
RC_i = w_i × MRC_i = w_i × (Σw)_i / σ_p
```

风险平价约束：
```
RC_i = RC_j,  ∀ i,j
```

等价于求解：
```
w_i × (Σw)_i = w_j × (Σw)_j
```

## 二、优化问题形式化

风险平价可转为无约束凸优化（Spinu 2013 公式）：

```
min f(x) = ½ xᵀΣx - Σ b_i·ln(x_i)

其中:
  x_i = w_i (原始权重变量)
  b_i = 风险预算 (risk budget)
  Σ   = 协方差矩阵
```

### 梯度

```
∇f(x) = Σx - b/x

其中 b/x 表示逐元素除法: (b_1/x_1, ..., b_n/x_n)
```

### Hessian

```
H(x) = Σ + diag(b/x²)

其中 diag(b/x²) = 对角矩阵, 对角元素为 b_i/x_i²
```

Hessian 的正定性保证：Σ 是半正定（协方差矩阵性质），diag(b/x²) 当 x>0 时严格正定。所以 H(x) 在整个可行域内**严格正定**，问题有唯一全局最优解。

## 三、Newton 法求解步骤

```
输入: Σ (协方差矩阵), budgets (风险预算向量), tol (收敛容差), max_iter
输出: w* (最优权重向量)

1. 初始化: x_0 = [1, 1, ..., 1]
2. for k = 0, 1, 2, ...:
    a. 计算梯度: g = Σ·x - budgets/x
    b. 检查收敛: if max(|RC_i - b_i|) < tol → 输出 x / sum(x)
    c. 计算 Hessian: H = Σ + diag(budgets/x²)
    d. 求解 Newton 方向: Δx = -H^{-1}·g
    e. Line search (回溯): 找到 α 使得:
       - x + α·Δx > 0 (非负约束)
       - f(x + α·Δx) < f(x) (充分下降)
    f. 更新: x = x + α·Δx
3. 归一化: w* = x / sum(x)
```

### Line search 细节

Newton 方向本身是好的下降方向，但可能让 x 变成负值。line search 的作用：
1. 从 α=1 开始
2. 如果 x + α·Δx 有负分量 → α = 0.5·α
3. 如果 f(x + α·Δx) ≥ f(x) → α = 0.5·α
4. 重复直到满足约束

防守性退出条件：
- Hessian 奇异 → 无法求解 Newton 方向 → break
- α 过小（< 1e-10）→ 认为已收敛
- max_iter 达到上限 → 返回当前解

## 四、与梯度下降的对比

| 维度 | Newton 法 | 梯度下降 |
|------|----------|---------|
| 收敛速度 | 二次收敛（近最优时极快） | 线性收敛 |
| 每次迭代成本 | O(n³)（矩阵求逆） | O(n²) |
| 对 Hessian 的要求 | 需要正定，奇异则失败 | 无 |
| 小规模 (n<20) | 推荐 | 也可 |
| 大规模 (n>100) | 求逆太贵 | 推荐或使用拟牛顿 |
| 风险平价适用性 | n≤10, 经典选择 | n>10, 备选 |

风险平价组合通常 ≤10 个资产类别，Newton 法是最合适的选择。

## 五、实现要点与常见陷阱

### 1. Hessian 调节（确保数值稳定性）

协方差矩阵的特征值可能极小（高度相关的资产），导致 Hessian 接近奇异。实践中加一个小的对角调节：

```
H_reg = H + λ·I,  λ = 1e-8 ~ 1e-6
```

### 2. 初始值敏感性

初始 x_0 应正且与 budgets 成比例：

```
x_0 = budgets / budgets.sum()  或  x_0 = [1/n, ..., 1/n]
```

用等权重初始化更稳健，不易落入边界。

### 3. 收敛判据选择

两种常用判据：
- **梯度范数**: `||g||_∞ < tol`（不够直观）
- **风险贡献偏差**: `max_i |RC_i/RC̄ - 1| < tol`（推荐）

后者直接对应风险平价的业务含义。

### 4. 协方差矩阵估计的影响

Newton 法本身是精确的，但输入 Σ 的质量决定了输出权重的质量：
- 用样本协方差 → 权重对估计窗口长度敏感
- 用收缩估计（Ledoit-Wolf）→ 更稳定
- 用 CNN/GARCH 预测 vol + 常数 corr → 快速但可能不准确

### 5. 权重上下界约束

基础 Newton 法只有 `w_i ≥ 0`。实际应用中需要：
- 上限约束 `w_i ≤ u_i`（如单 ETF ≤ 20%）
- 整数手数约束（ETF 按手交易）
- 换手率约束（每次调仓不超过 x%）

这些约束可以通过以下方式处理：
- 投影法：在 Newton step 后投影到可行域
- 内点法：添加 log-barrier 项
- 交替方向法：在无约束解上做最小二乘投影

### 与全天候框架的关系

当前 parity.py 使用固定预算 `[0.4, 0.3, 0.15, 0.15]`。在全天候框架中，预算应从状态识别层动态生成：

```
宏观状态 Q → 预算映射器 → b(Q) → Newton 求解器 → w*(b, Σ)
```

这意味着每次再平衡时，Newton 求解器用**不同的 budget** 但**相同的算法**重新优化。Newton 法的快速收敛在此场景下是优势——每次调仓只需要 5-10 次 Newton 迭代即可收敛。

## 附录：术语逐一解释

### 协方差矩阵 Σ (Covariance Matrix)

**白话**：一张表，记录每对资产之间"一起涨跌"的程度。

假设有 3 个资产 A、B、C。Σ 是一个 3×3 矩阵：

|   | A   | B   | C   |
|---|-----|-----|-----|
| A | A自己的方差 | A和B的协方差 | A和C的协方差 |
| B | B和A的协方差 | B自己的方差 | B和C的协方差 |
| C | C和A的协方差 | C和B的协方差 | C自己的方差 |

- 对角线上是**方差**（一个资产自己的波动大小）
- 非对角线上是**协方差**（两个资产一起涨跌的程度）
- 协方差为正 → 同涨同跌；协方差为负 → 一个涨另一个跌

**为什么重要**：组合风险不是各资产风险的简单相加，要看它们之间的相关性。两个高风险资产如果负相关，组合起来风险反而小。

---

### 组合波动率 σ_p (Portfolio Volatility)

**白话**：整个投资组合的"颠簸程度"，数字越大说明净值涨跌越剧烈。

```
σ_p = sqrt(wᵀ Σ w)
```

展开就是：每个资产的权重 × 每个资产的波动 × 资产间的相关性，全部揉在一起算出一个数。

---

### 边际风险贡献 MRC (Marginal Risk Contribution)

**白话**：如果我把某个资产加仓 1%，组合总风险会增加多少。

```
MRC_i = (Σw)_i / σ_p
```

即：该资产与所有其他资产的协方差加权求和，再除以组合总波动。

**举例**：组合里有股票和债券。股票的 MRC=0.15，债券的 MRC=0.03。意思是多买 1% 的股票会让组合风险增加 0.15%，而多买 1% 债券只增加 0.03%。股票对风险的"边际贡献"远大于债券。

---

### 风险贡献 RC (Risk Contribution)

**白话**：某个资产实际贡献了多少风险给整个组合。等于该资产的权重 × 它的 MRC。

```
RC_i = w_i × MRC_i
```

**举例**：股票占组合 60%，MRC=0.15，RC=60%×0.15=9%。债券占 40%，MRC=0.03，RC=40%×0.03=1.2%。虽然股票权重只比债券多一半，但它贡献的风险是债券的 7.5 倍——因为股票本身就比债券波动大得多。

**所有资产的 RC 加起来 = 组合总风险 σ_p**。

---

### 风险平价 (Risk Parity)

**白话**：让每个资产贡献的**风险**一样多，而不是**钱**一样多。

- 等权重：每个资产投一样多的钱（各 25%）。这是钱的平等。
- 风险平价：每个资产贡献一样的风险。这是风险的平等。

**为什么不用等权重**：如果组合是 50% 股票 + 50% 债券，股票贡献了 ~90% 的风险，债券只贡献 ~10%。组合的涨跌几乎完全由股票决定，债券的"分散风险"作用被稀释了。

风险平价通过调整权重（多配债券、少配股票），让两者的 RC 相等，真正做到风险分散。

---

### 风险预算 b_i (Risk Budget)

**白话**：你"允许"每个资产贡献多少比例的风险。

比如 `b = [0.4, 0.3, 0.15, 0.15]` 意思是：
- 资产 1：允许贡献 40% 的组合风险
- 资产 2：30%
- 资产 3：15%
- 资产 4：15%

纯风险平价要求所有 b_i 相等（各 25%）。但在全天候策略中，b_i 随宏观环境动态变化——通胀高的时期，你可以给商品多一点风险预算。

---

### 梯度 ∇f (Gradient)

**白话**：函数在每个位置上"最陡的上升方向"。就像站在山坡上，梯度指向最陡的上坡方向。

```
∇f(x) = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ]
```

- 梯度的每个分量告诉你：只改变这一个变量时，函数值变化多快
- 在优化中，**负梯度方向**（-∇f）就是"最陡的下坡方向"——往那走能最快减小函数值

**举例**：f(x)=x²，在 x=3 处，∇f=6。说明 x 增加一点点，f 会增加 6 倍那么多。往反方向（减小 x）走就能让 f 变小。

---

### Hessian 矩阵 H (Hessian Matrix)

**白话**：梯度的梯度。如果说梯度告诉你坡度有多陡，Hessian 告诉你坡度的**变化率**——坡是越来越陡还是越来越缓。

```
H_ij = ∂²f / ∂x_i∂x_j
```

这是一个 n×n 矩阵，每个元素是 f 对两个变量的二阶偏导。

**在优化中的作用**：
- 梯度下降只看一阶信息（坡的方向），所以只能线性收敛
- Newton 法看二阶信息（坡的弯曲程度），能预测最优解在哪里，所以收敛快得多

---

### 正定性 / 半正定性 (Positive Definiteness)

**白话**：一个矩阵是否"处处往上弯"。正定矩阵对应的二次型永远是正数（除了在零点）。

- **正定**：xᵀHx > 0 对所有非零 x。函数是严格凸的，只有一个"碗底"——唯一最小点。
- **半正定**：xᵀHx ≥ 0。函数是凸的，但可能有平坦区域——多个最小点。

**在风险平价中**：Hessian 严格正定 = 只有一个全局最优解，Newton 法一定收敛到它。这是为什么 Newton 法适合风险平价的理论基础。

---

### 奇异矩阵 (Singular Matrix)

**白话**：一个矩阵没有逆矩阵。相当于"除以零"的矩阵版。

- 可逆矩阵：存在 A⁻¹，使得 A·A⁻¹ = I（单位矩阵）
- 奇异矩阵：det(A)=0，逆矩阵不存在

**为什么 Hessian 会奇异**：当两个资产几乎完全同涨同跌（比如沪深 300 ETF 和上证 50 ETF），它们的信息几乎完全重叠。此时 Σ 的某个特征值接近零，H 的对应特征值也接近零，H 变得不可逆。

**对策**：加对角调节 H_reg = H + λ·I，相当于给每个变量都加了一点点独立的"场"，让矩阵可逆。

---

### Newton 法 (Newton's Method)

**白话**：一种"聪明"的优化方法。想象你在山谷里找最低点：
- 梯度下降：只看脚下的坡度，一步一步往下走，到了谷底附近要碎步挪
- Newton 法：看了脚下的曲率，直接"跳"到预测的最低点，一步到位（如果不准就再来一步）

数学上，Newton 法用一个二次函数来近似原函数，然后直接求二次函数的最小值作为下一步。因为二次函数的形状由 Hessian 决定，所以 Newton 法利用了二阶信息。

---

### Line Search（线搜索 / 回溯）

**白话**：Newton 法预测的"跳跃"可能太大，直接跳到了非法区域（比如权重变负数）。Line search 把步长缩小，保证每步都合法且确实在减小目标函数。

```
x_new = x + α × Δx
```

α=1 是全步长（Newton 预测的最优跳跃）。如果跳出去不合法，就 α=0.5, 0.25, 0.125... 往回缩，直到合理为止。

---

### 特征值 (Eigenvalue)

**白话**：矩阵的内在属性。一个 n×n 协方差矩阵的 n 个特征值代表数据在 n 个独立方向上的方差（分散程度）。

- 特征值大 → 数据在这个方向上很分散
- 特征值接近零 → 数据在这个方向上几乎不变化
- 有特征值为零 → 矩阵奇异

**在风险平价中**：如果 Σ 有接近零的特征值，说明有些资产是冗余的（几乎线性相关）。这会传导到 H 使 Newton 步不可靠，需要对角调节。

---

### Ledoit-Wolf 收缩估计 (Shrinkage Estimator)

**白话**：一种"更稳"的协方差矩阵估计方法。

- 样本协方差：直接用历史数据算。对数据敏感，历史窗口变一点结果就变。
- Ledoit-Wolf：把样本协方差向一个"目标矩阵"（比如单位矩阵）"收缩"一点。

```
Σ_LW = δ × Σ_target + (1-δ) × Σ_sample
```

δ 是收缩强度，由数据自动决定。相当于在"完全信数据"和"假设资产都独立"之间取折中，结果更稳定。

---

### 投影法 (Projection Method)

**白话**：处理约束的一种技巧。算出无约束的最优解后，如果它违反了某个约束，就把它"投影"回可行域的最近点。

**举例**：Newton 法算出某资产权重应该是 35%，但规则说单资产不能超过 20%。就把它从 35% 投影到 20%（最近的可行的点）。

---

### 内点法 / Log-Barrier (Interior Point Method)

**白话**：通过修改目标函数来"软性"约束权重。

```
新目标 = 原目标 - μ × ln(x_i)
```

ln(x) 在 x→0 时趋近 -∞，所以目标函数在权重接近零时急剧增大。优化器会自动远离边界，保持在可行域内部。μ 控制"墙有多硬"。

---

### O(n³) / O(n²)（算法复杂度）

**白话**：衡量计算量随问题规模增长的速度。

- O(n²)：资产数量翻倍，计算量翻 4 倍
- O(n³)：资产数量翻倍，计算量翻 8 倍

Newton 法的矩阵求逆是 O(n³)，所以 n=100 时比 n=10 慢 1000 倍。但风险平价通常 n≤10，O(n³) 完全不是问题。

---

### 收敛容差 tol (Tolerance)

**白话**："够好了"的门槛。数值优化不可能算到完全精确，tol=1e-6 的意思是：当每个资产的风险贡献偏差小于百万分之一时，就认为已经达到风险平价，停止迭代。

---

### 归一化 (Normalization)

**白话**：把权重缩放到总和为 1（即 100%）。

Newton 法求解出来的 x 是最优的比例，但不一定加起来是 1。最后一步 `w* = x / sum(x)` 就是归一化，保证权重加起来等于全部资金。权重比例不变，只是缩放。
