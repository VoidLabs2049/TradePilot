▸ 8 个可迁移技术/工程模式
  • 1. 三专家 MWU 动态权重 — Top-K收益做reward，动态调整信任度。多信号融合的通用范式。
  • 2. CVXPY MILP 离散优化 — 整数手数 + 单只上限 + 现金缓冲。A股100股倍数的标准解法。
  • 3. 五层风控过滤器链 — 振幅→持仓上限→禁回购→冷却→成本。不是单一止损，而是多层防御。
  • 4. 公共新闻行业路由 — embedding相似度 + 行业映射 + 权重折扣。NLP信号质量控制。
  • 5. 筹码分析量化落地 — Tushare cyq_perf → 集中度/获利比/成本偏离。筹码理论的工程实现。
  • 6. MWU 健康检查日志 — 非零率/reward/dw 详细诊断。防止黑箱的工程纪律。
  • 7. 卖出冷却 + 当日禁回购 — 制度性防频繁交易。A股 T+1 最佳实践。
  • 8. P0-P4 分阶段增强路线 — 规格冻结→离线验证→准入评审→主链接入。大系统迭代的标准方法论。
⚠ 需要关注的问题
  • ▪ 筹码集中度系数为负（-0.30），与"单峰密集=重仓"建议相反 — 可能是有意设计或需要验证
  • ▪ Tushare Token 和 OpenAI Key 明文在 config 中 — 需脱敏/环境变量
  • ▪ infra.py 1428行单文件 — 需模块化拆分
  • ▪ NLP 依赖外部 OpenAI API — 成本+延迟+稳定性单点风险
  • ▪ 无单元测试 — 相比 RV_Transformer 有7个测试
